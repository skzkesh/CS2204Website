Javascript

Event Handler

On my website, I apply two types of event handlers: HTML and Javascript. I assigned the first event handler onclick directly in the Javascript and connected as seen in visit.js line 1 to line 37. The event handler is assigned to the button element. When the button is clicked, the validateForm function in JavaScript will be called, and validation will be performed based on the input given. I also use onclick on my apply page, but I set the event handler directly in the HTML this time. When the button in each form is clicked, it will execute the getRank(this) function, which will append the rank to the table. This code can be seen in every form in my apply.html. The getRank function can be seen in lines 110-147 of the apply.js.

Let and Const

I use the const messages variable for the random messages shown on the program page, as seen in program.js lines 1-3. I also put the video sources, controls, and muted in a const video on lines 29-48. The reason for using const is for the value to remain constant throughout the execution. Const is immutable, which means the value cannot be changed or reassigned. I put those properties in const because they contain information that should stay the same. I also use let inside of the getRank function (apply page line 111 to line 115) because the variable declared with let is limited to the function scope. So, it helps me to prevent the accidental use of variables before they are adequately assigned. It can be seen in the apply page line 85 to line 95.

Class

I created a dictionary of college and majors for each class. Then, I made a class called Major, which will point to the class code of the chosen button. The class will return the major and college of the given class code. I can append that information to my table directly. The class helps me to increase code readability and enhance effectiveness since I do not have to write multiple similar codes. Instead, I need to access them just by using their ID.

Home Page (homePage.css)

For the welcome block, I use text-align center and some other features to control the view (line 1 -7). The basic information block and map are divided using multi column layout where I give each block 50% width, display block, and float left (line 8-24). I used overflow: auto; (line 11) for the basic information so users can still scroll down the overlap content when they minimize the screen. My webpage also helps user to see the location image more clearly by zooming it in when they hover on the image. I use transform: scale(1.3) with border-radius set in #location_image:hover to enable this effect (lines 40-43). This effect can still work properly when we minimize the screen.

Apply (apply.css)(printApply.css)

Before the hover, the display of college lists are hidden, but after the hover, it become a display block (example, line 38-49). To make the color stay white after we move from the heading, I use college:has(~ .class:hover) with the background color white (line 55). All the headings are set with the same width, which is 30% (line 22). All colleges will have 90% width when we open them (example, line 65). All the majors have display inline and are floating left with some border-radius. I set the image display to block. The border left, right, and bottom are 1 px solid black. The table has the same layout as the previous table. The submit button will change appearance when we hover (line 174-186). I also create print versions only with header, footer, apply, and result section. All the others element are in display: none.
